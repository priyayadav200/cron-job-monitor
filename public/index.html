Six core capabilities designed for teams that run critical scheduled tasks.

Health Scoring

Each job gets a 0-100 health score from its overall success rate (40%) and recent execution trend (60%). Scores map to healthy, warning, degraded, and critical.

Failure Detection

Tracks non-zero exit codes and captures error output per run. Surfaces failure patterns with per-job failure counts and incident tracking.

Missed Run Detection

Configurable grace periods per job. If a ping doesn't arrive within the expected window, CronWatch flags it as missed and triggers an alert.

Duration Analytics

Per-job avg, min, max, and P95 execution durations with daily trend charts. Spot performance regressions before they become outages.

Ping Integration

Add a single curl call to your script. CronWatch provides a unique URL per job — when the ping arrives, the run is recorded. One line of code.

Alert Routing

Route failure notifications to email, Slack, or any webhook endpoint. Configure rules per job — alert on failure, missed check-in, or duration anomaly.

Create a monitor

02

Add one line to your script

Append a curl call at the end of your cron job. CronWatch records the ping with timestamp and duration.
