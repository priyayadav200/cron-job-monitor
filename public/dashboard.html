Integrate CronWatch with your cron jobs

Add a single line to your script and CronWatch will track every execution, detect failures, and alert you when something goes wrong.

How Ping Monitoring Works

1
Create a monitor

Add your cron job with its schedule. You'll get a unique ping URL.

2
Add ping to your script

Call the ping URL at the end of your cron job. One line of code.

3
CronWatch tracks it

If the ping doesn't arrive on time, CronWatch flags it as a missed run and alerts you.

Ping URL Format

Endpoint
GET  https://your-host.com/ping/{job_id}

Replace {job_id} with your monitor's Job ID (visible in monitor details).

Integration Examples

cURL
# Add to the end of your cron job script
curl -fsS --retry 3 https://your-host.com/ping/YOUR_JOB_ID

# With timing (reports execution duration)
START=$(date +%s)
# ... your job logic here ...
DURATION=$(( $(date +%s) - START ))
curl -fsS "https://your-host.com/ping/YOUR_JOB_ID?duration=$DURATION"

# Report failure
curl -fsS "https://your-host.com/ping/YOUR_JOB_ID?state=fail&msg=backup+failed"

Crontab Example

crontab -e
# Database backup every day at 2 AM — pings CronWatch on completion
0 2 * * * /usr/local/bin/backup.sh && curl -fsS https://your-host.com/ping/YOUR_JOB_ID

Query Parameters

ParameterTypeDescription
statestring"complete" (default), "fail", or "run" (start signal)
durationintegerExecution duration in seconds
msgstringOptional message or error output

Alert Rules

Job Failure

Alert immediately when a job exits with a non-zero code

Missed Check-in

Alert when a job doesn't ping within its expected schedule + grace period

Duration Anomaly

Alert when execution takes 2x longer than the average duration

Health Score Drop

Alert when a monitor's health drops below 85%
